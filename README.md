# 🧑 人生模拟器 · 命运模型 v2

> 把你自己「丢进」中国的宏观经济里，看不同的人生选择会带来怎样的终局。
>
> 也可以对任何名人做「人生剧本复盘」—— 算一算他/她的**正确选择率**，找出哪些分岔路口其实可以选得更好。

---

## ✨ 功能一览

| 模块 | 说明 |
|------|------|
| 🧑 **个人画像** | 学历 / 智商 / 野心 / 自律 / 逆商 / 社交 / 风险偏好 / 健康基线 / 家庭背景 |
| 🇨🇳 **宏观经济** | GDP / 通胀 / 失业 / 工资 / 房价 / AI 渗透 / 老龄化 / 教育回报率 / 地缘风险 … |
| 🔄 **5 种策略对比** | 稳定保守 / 努力奋斗 / 激进冒险 / 平衡生活 / 躺平低欲望 |
| 📊 **6 大曲线图** | 财富、职业、健康、幸福、AI 替代风险、宏观指标 |
| 🌟 **名人人生剧本分析** | 预设周涛 / 董明珠 / 马云 / 张一鸣模板，也可以自定义任何名人 |
| 📐 **正确选择率** | 对每个关键人生分岔点打分，给出改进建议 |

## 🚀 快速开始

### 方式一：网页版（推荐 ⭐）

直接双击打开 `index.html`，或用简易 HTTP 服务：

```bash
# 方式 1：用 shell 脚本
./start_web.sh

# 方式 2：手动起 Python 服务
python3 -m http.server 8080
# 浏览器访问 http://localhost:8080/index.html
```

> 💡 用 HTTP 服务比直接 `file://` 打开更稳，能避免浏览器对本地 JS 的安全限制。

### 方式二：命令行版（Python）

```bash
# 跑一个叫「小明」的人的一生
python3 run_demo.py

# 跑周涛的人生剧本分析（正确选择率）
python3 simulate_zhoutao.py
```

## 🧭 网页界面使用指南

网页从上到下分为 **3 大区域**：

### ① 左侧 · 人物画像面板

填一个人的基础信息：

- **基本信息**：姓名、出生年份、最高学历（文盲→博士）、出身家庭（贫困→富裕）
- **个人特质**（滑块 0–100）：智商、野心、自律、逆商、社交能力、风险偏好
- **健康与医疗**：健康基线、就医条件、遇到好医生的运气、慢性病/遗传风险
- **人生策略**：当前选中的主策略（后续中间按钮可同时跑 5 种对比）

### ② 中间 · 图表与分析区

- **KPI 卡片**：终局财富、平均幸福度、失业年数、AI 替代风险
- **按钮**：
  - `▶ 开始模拟一生` — 用当前参数跑一次
  - `🔀 对同一人跑 5 种策略做对比` — 一次出 5 条曲线对比
  - `🎓 测学历：文盲→硕士` — 看不同学历对终局的影响
- **6 张曲线图**：财富积累、职业等级、健康状况、幸福感、AI 替代风险、宏观经济指标
- **📜 关键事件时间线**：一生中的重大节点（如失业、患病、升职、财富爆发等）

### ③ 右侧 · 宏观经济 + 策略对比

- **时代场景预设**：中国 2025（新常态）/ 中国 2035（AI 高速 + 老龄化）/ 中国 2045（低增长 + 深度老龄化）/ 中国 1995（高速增长黄金期）/ 自定义
- **自定义宏观滑块**：你可以拨动任何宏观参数（GDP、通胀、失业率、AI 渗透率、教育回报率等）即时看影响
- **策略对比面板**：5 种策略的终局结果并排展示（财富 / 幸福 / 失业年数）

## 🌟 名人人生剧本分析模块

> 页面最下方的「**名人人生剧本分析**」区域，可以对任何人做「反事实复盘」。

### 使用方法

1. **选择预设模板**：周涛 / 董明珠 / 马云 / 张一鸣，或选「自定义」
2. **填/改 基础画像**：调整该名人的智商、野心、健康等参数
3. **设计关键人生分岔点**：
   - 每个岔路口有两条路：⭐ **实际选择的路** vs 💭 **他没走的路**
   - 每条路可以设定不同的「**人生策略**」和「**行业**」
   - 比如：30 岁 · 跳央视 vs 留守北京台
4. 点击「**▶ 开始分析 · 计算正确选择率**」

### 输出包含

- **🏆 所有路径排名** — 按综合得分排序，标出全局最优 / 最差
- **📊 每个岔路口的选择复盘** — 每一条路的得分、分差、判断「选对 / 选错 / 差不多」
- **💡 综合建议** — 正确选择率、改进建议、全局最优路径总结

### 综合得分公式

```
综合得分 = 幸福感 × 40% + 职业等级 × 30% + 财富 × 20% + 健康 × 10%
```

## 🧠 模型核心原理

### 1. 个人画像（Person）

每个人由 12 个参数刻画：

| 参数 | 含义 | 作用 |
|------|------|------|
| `birth_year` | 出生年份 | 决定他经历的时代（中国经济哪个阶段） |
| `education` | 最高学历 | 影响**人力资本**和**文凭信号** |
| `iq` | 智商 / 学习能力 | 职业上升 + AI 风险对冲 |
| `ambition` | 野心 | 职业动机强度 |
| `discipline` | 自律 | 平滑积累的关键 |
| `resilience` | 逆商 | 危机后的恢复能力 |
| `social_skill` | 社交能力 | 社会资本积累 |
| `risk_tolerance` | 风险偏好 | 冒险策略的收益/代价 |
| `health_baseline` | 健康基线 | 健康曲线起点 |
| `medical_access` | 就医条件 | 健康恢复速度 |
| `heredity_risk` | 遗传风险 | 老年健康下降速度 |
| `family_*` | 家庭背景 | 初始财富 / 社会资本 |

### 2. 宏观经济引擎（MacroEngine）

每次模拟按年龄一年年推进，每一年的宏观环境由这些变量决定：

| 变量 | 符号 | 说明 |
|------|------|------|
| **GDP 增速** | `gdp_growth` | 决定整体机会 |
| **通胀率** | `inflation` | 侵蚀财富的购买力 |
| **失业率** | `unemployment` | 个人失业概率的基线 |
| **工资增速** | `wage_growth` | 整体收入的上涨 |
| **房价增速** | `house_price_growth` | 财富的重要组成 |
| **AI 渗透率** | `ai_adoption` | 高学历 + 高 IQ 者受益；低学历者被替代 |
| **老龄化** | `aging` | 医疗压力 & 养老金压力 |
| **教育回报率** | `edu_return` | 高学历的溢价 |
| **学历贬值** | `edu_inflation` | 相同文凭在不同时代的含金量 |
| **贫富差距** | `gini` | 冒险策略的天花板 |
| **政策稳定性** | `policy_stability` | 大起大落的概率 |
| **地缘风险** | `geopolitical_risk` | 外部危机概率 |

**核心机制**：个人的职业/收入/财富不是独立的，而是**被宏观环境乘上一个系数**。
比如 1990-2010 年中国高增长时代，"努力奋斗"策略的回报特别高；而 2030 年后的低增长 + 高 AI 时代，"平衡生活"可能更好。

### 3. 人生策略（Strategy）

| 策略 | 努力强度 | 职业向上流动 | 储蓄率 | 投资风险 | 持续学习 |
|------|---------|-------------|--------|---------|---------|
| **稳定保守** | 0.8 | 0.4 | 0.25 | 低 | 低 |
| **努力奋斗** | **1.0** | **0.9** | **0.20** | 中 | **高** |
| **激进冒险** | 0.95 | 1.0 | 0.10 | **很高** | 高 |
| **平衡生活** | 0.85 | 0.6 | 0.18 | 中 | 中 |
| **躺平低欲望** | 0.4 | 0.1 | 0.15 | 很低 | 很低 |

策略 × 宏观 × 个人特质 = **每年的收入、消费、财富积累、职业等级、健康损耗、幸福感** 的联立方程。

### 4. 每年的模拟循环

对每一年（从 18 岁 → 75 岁）：

```
1. 取当年的宏观参数（GDP / 失业 / AI 渗透 ...）
2. 计算当年职业等级：
   career_level += 策略强度 × 个人特质 × 宏观景气度 − 年龄衰退因子
3. 计算当年收入：
   income = 基准工资 × 职业等级 × 工资增速 × (1 − 失业风险)
4. 计算财富：
   wealth += income × 储蓄率 + 投资收益
   投资收益 = wealth × (利率 + 投资策略 × 宏观波动率)
5. 计算健康：
   health += 医疗投入 − (年龄自然衰退 + 工作压力 + 随机健康事件)
6. 计算幸福感：
   happiness = f(收入, 财富, 健康, 社交资本, 工作压力, 安全感)
7. 判断 AI 替代风险：
   ai_risk = AI 渗透率 × (1 − 学历/人力资本对冲系数)
8. 生成关键事件（如 "35 岁跳槽成功"、"42 岁中年危机被裁员" ...）
```

## 📁 项目文件结构

```
.
├── index.html              # 网页版主界面（HTML + 样式 + 模块布局）
├── fate_model.js           # 前端命运模型（完全自洽，无需后端）
│                            # 包含：Person 构建 / MacroEngine / FateSimulator / 图表渲染
├── fate_model.py           # 后端命运模型（Python 版，逻辑与 JS 版一致）
│                            # 包含：Person / Country / MacroEngine / FateSimulator
├── simulate_zhoutao.py     # 周涛/名人人生剧本分析（命令行版）
│                            # 输入：分岔点配置 → 输出：路径排名 + 正确选择率
├── run_demo.py             # 快速演示脚本
├── start_web.sh            # 一键启动本地 HTTP 服务（访问 index.html）
└── .gitignore              # 忽略临时 JSON 结果、__pycache__ 等
```

### 前端核心模块（`fate_model.js`）

| 模块 | 功能 |
|------|------|
| `EDUCATION_LEVELS` | 9 个学历等级的人力资本 & 文凭信号 |
| `FAMILY_PRESETS` | 4 档家庭背景（贫困/普通/中产/富裕） |
| `STRATEGIES` | 5 种人生策略的参数矩阵 |
| `MACRO_SCENES` | 5 个时代场景的宏观参数预设 |
| `buildPerson()` | 从表单参数构造一个 Person 对象 |
| `MacroEngine` | 按年份推进的宏观经济模拟引擎 |
| `FateSimulator` | 命运模拟主循环（每年收入/财富/健康/幸福） |
| `ensureChart()` | Chart.js 封装：画所有曲线图 |

### 后端核心模块（`fate_model.py`）

| 模块 | 功能 |
|------|------|
| `Person` (dataclass) | 人物画像数据结构 |
| `Country` | 国家/时代预设（中国 1995 / 2025 / 2035 / 2045） |
| `MacroEngine` | 宏观经济引擎 |
| `FateSimulator` | 命运模拟主循环（与 JS 版逻辑一致） |
| `compare_all()` | 多人物 × 多国家对比函数 |

> 💡 JS 版和 Python 版的模型**核心逻辑一致**，参数体系相同，但因浮点数和随机数生成器不同，数值会略有差异。

## 🎯 模型的意义与局限

### ✅ 模型能做什么

1. **反事实思考** — "如果当年我选了另一条路会怎样？" — 这是模型最大的价值。它强迫你把直觉变成可比较的参数。
2. **时代红利的可视化** — 同一个人，放在 1995 年的中国 vs 2035 年的中国，最优策略完全不同。
3. **策略匹配** — "努力奋斗"在高增长时代是王者；在低增长 + 高 AI 时代，"平衡生活"可能综合得分更高。
4. **名人故事的去魅** — 很多"成功人士"，把他的选择换成保守路径再跑一遍，你会发现有些选择其实是时代红利而非能力。

### ⚠️ 模型不能做什么

1. **不是算命** — 所有数字都是相对大小、方向性的比较，不是真实预测。
2. **不包含运气的极端值** — 模型里有随机数，但没有"中彩票/遇到贵人/一夜暴富"这种量级的突变。真实人生的方差比模型大得多。
3. **不包含社会网络的复杂效应** — 模型里的"社交资本"是一个简化的累积值，没有模拟人脉网络的指数增长。
4. **不包含感情 / 婚姻的非线性** — 婚姻是真实人生里改变命运最大的变量之一，模型目前只简化为"社会资本 + 幸福加成"。

> **一句话**：用它来做"方向性的反事实思考"—— 比如"XX 和 YY 哪个更重要"、"策略 A 比策略 B 好多少"，不要用它算具体的数字。

## 🛠 开发者：如何扩展

### 添加一个新的名人模板

打开 `fate_model.js`，找到 `CELEBRITY_TEMPLATES` 对象，在后面追加：

```javascript
"你的模板key": {
  name: "姓名",
  birth_year: 1975,
  education: "本科",
  field: "互联网/科技",
  iq: 85, ambition: 90, discipline: 85, resilience: 80,
  social: 80, risk: 75, health0: 75,
  family_wealth: 50, family_social: 50,
  forks: [
    { title: "25 岁 · 去大厂还是创业",
      real_name: "去腾讯做程序员", real_strategy: "hard", real_field: "互联网/科技",
      alt_name: "自己创业做网站", alt_strategy: "radical", alt_field: "互联网/科技" },
    // ... 可以加 N 个分岔点
  ],
},
```

然后在 `index.html` 的 `<select id="celeb-template">` 里加一个 `<option>`。

### 调整模型参数

- 想改宏观预设 → 改 `MACRO_SCENES`
- 想改策略 → 改 `STRATEGIES`
- 想改学历溢价 → 改 `EDUCATION_LEVELS`
- 想改每年的模拟逻辑 → 改 `FateSimulator.run()` 内部循环

### 改 UI 样式

- 打开 `index.html`，所有 CSS 都在 `<style>` 标签里
- 图表用的是 **Chart.js**（通过 CDN 加载，无需安装）

## 📝 关于本项目

- **初版动机**：很多人讨论命运，但很少人把「个人选择」和「时代背景」放到一个模型里做对比。这个项目试图提供一个可以反复把玩的框架。
- **开发语言**：纯 HTML + CSS + JavaScript（前端），以及 Python 3（后端/命令行）。前端**不需要任何构建工具**，双击就能用。
- **依赖**：前端只用了 Chart.js（CDN 加载），后端是纯标准库。
